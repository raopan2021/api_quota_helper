## 第一章：Kotlin 基础（对比 JavaScript）

### 1.1 变量声明

```kotlin
// val = 常量（不可变），类似 const
val name = "API Helper"

// var = 变量（可变更），类似 let
var count = 0
count = 1

// 类型推断（自动识别）
val message = "Hello"  // String 类型

// 显式类型声明
val age: Int = 25
val price: Double = 19.99
val isActive: Boolean = true
```

**对比 JavaScript：**
```javascript
const name = "API Helper"
let count = 0
const age = 25
const price = 19.99
const isActive = true
```

### 1.2 空安全（重要！）

Kotlin 是空安全的语言：

```kotlin
// 不能直接赋 null
val name: String = null  // ❌ 编译错误

// 使用 ? 表示可空类型
val name: String? = null  // ✅

// 安全调用操作符 ?. （类似可选链）
val length = name?.length  // 如果 name 是 null，返回 null

// Elvis 操作符 ?: （类似 ??）
val length = name?.length ?: 0  // 如果 name 是 null，返回 0

// 非空断言 !! （慎用）
val length = name!!.length  // 如果 name 是 null，抛出异常
```

**对比 JavaScript：**
```javascript
const name = null
const length = name?.length ?? 0  // 原生也支持可选链
```

### 1.3 函数

```kotlin
// 基本函数
fun greet(name: String): String {
    return "Hello, $name"
}

// 单表达式函数（简写）
fun greet(name: String) = "Hello, $name"

// 默认参数
fun greet(name: String = "World") = "Hello, $name"

// 命名参数（调用时指定参数名）
greet(name = "API", age = 25)
```

**对比 JavaScript：**
```javascript
const greet = (name = "World") => `Hello, ${name}`
```

### 1.4 条件与循环

```kotlin
// if 是表达式（有返回值）
val result = if (count > 0) "正数" else "非正数"

// when 替代 switch
when (status) {
    1 -> "进行中"
    2 -> "已完成"
    else -> "未知"
}

// for 循环
for (item in list) {
    println(item)
}

// 带索引的循环
for ((index, item) in list.withIndex()) {
    println("$index: $item")
}

// 区间
for (i in 0..10) { }  // 0 到 10（包含）
for (i in 0 until 10) { }  // 0 到 9（不包含）
```

**对比 JavaScript：**
```javascript
const result = count > 0 ? "正数" : "非正数"
switch (status) {
    case 1: return "进行中"
    default: return "未知"
}
for (const item of list) { }
for (const [index, item] of list.entries()) { }
```

### 1.5 类（Class）

```kotlin
// 定义类
class User(val name: String, var age: Int) {
    // 主构造函数直接写在类头
    
    // 方法
    fun greet() = "Hello, I'm $name"
}

// 继承
open class Animal(val name: String) {
    open fun sound() = "..."
}

class Dog(name: String) : Animal(name) {
    override fun sound() = "汪汪"
}

// 数据类（自动生成 equals, hashCode, copy 等）
data class UserAccount(
    val id: String,
    val username: String,
    val token: String
)
```

**对比 JavaScript：**
```javascript
class User {
    constructor(name, age) {
        this.name = name
        this.age = age
    }
    greet() { return `Hello, I'm ${this.name}` }
}
```

### 1.6 Lambda 表达式

```kotlin
// 基本 lambda
val sum: (Int, Int) -> Int = { a, b -> a + b }

// 单参数可用 it
list.filter { it > 0 }

// map / forEach
list.map { it * 2 }
list.forEach { println(it) }

// 返回 Unit（类似 void）
val printAll: (List<String>) -> Unit = { list ->
    list.forEach { println(it) }
}
```

**对比 JavaScript：**
```javascript
const sum = (a, b) => a + b
list.filter(x => x > 0)
list.map(x => x * 2)
```

## 第二章：Jetpack Compose（对比 Vue Composition API）

Compose 是 Android 的现代 UI 框架，**声明式**写法与 Vue Composition API 非常相似。

### 2.1 组件定义

```kotlin
// @Composable 注解标记这是 UI 组件
@Composable
fun Greeting(name: String) {
    Text(text = "Hello, $name!")
}

// 可组合函数可以嵌套其他 @Composable
@Composable
fun UserCard(user: User) {
    Card {
        Column {
            Text(user.name)
            Text(user.email)
        }
    }
}
```

**对比 Vue：**
```vue
<template>
    <div>Hello, {{ name }}!</div>
</template>

<script setup>
defineProps<{ name: string }>()
</script>
```

### 2.2 状态管理（核心！）

```kotlin
@Composable
fun Counter() {
    // remember 保存状态，跨重组保持
    var count by remember { mutableStateOf(0) }
    
    // 类似 ref
    var text by remember { mutableStateOf("") }
    
    Column {
        Text("计数: $count")
        Button(onClick = { count++ }) {
            Text("增加")
        }
    }
}
```

**对比 Vue：**
```vue
<script setup>
import { ref } from 'vue'
const count = ref(0)
const text = ref("")
</script>
```

### 2.3 状态提升（类似 props 传递）

```kotlin
// 父组件
@Composable
fun Parent() {
    var name by remember { mutableStateOf("API") }
    
    // 传递给子组件
    ChildComponent(
        name = name,
        onNameChange = { name = it }
    )
}

// 子组件接收 props
@Composable
fun ChildComponent(
    name: String,
    onNameChange: (String) -> Unit
) {
    OutlinedTextField(
        value = name,
        onValueChange = onNameChange
    )
}
```

**对比 Vue：**
```vue
<!-- Parent.vue -->
<script setup>
import { ref } from 'vue'
const name = ref("API")
</script>
<ChildComponent :name="name" @update="name = $event" />

<!-- Child.vue -->
<script setup>
defineProps<{ name: string }>()
const emit = defineEmits(['update'])
</script>
```

### 2.4 副作用（类似 watch / useEffect）

```kotlin
@Composable
fun MyComponent() {
    val data by viewModel.data.collectAsState()
    
    // LaunchedEffect = useEffect
    LaunchedEffect(key1 = userId) {
        // 相当于 useEffect(() => { ... }, [userId])
        viewModel.loadUser(userId)
    }
    
    // 定时器示例
    var time by remember { mutableStateOf(0) }
    LaunchedEffect(Unit) {
        while (true) {
            delay(1000)
            time++
        }
    }
}
```

**对比 Vue：**
```vue
<script setup>
import { ref, watch, onMounted } from 'vue'

watch(userId, () => {
    viewModel.loadUser(userId)
}, { immediate: true })

onMounted(() => {
    const timer = setInterval(() => time.value++, 1000)
    onUnmounted(() => clearInterval(timer))
})
</script>
```

### 2.5 布局组件（对应 HTML + CSS）

```kotlin
@Composable
fun MyLayout() {
    // Column = flex-direction: column
    Column(
        modifier = Modifier
            .fillMaxWidth()
            .padding(16.dp),
        horizontalAlignment = Alignment.CenterHorizontally,
        verticalArrangement = Arrangement.spacedBy(8.dp)
    ) {
        // Row = flex-direction: row
        Row(
            modifier = Modifier.fillMaxWidth(),
            horizontalArrangement = Arrangement.SpaceBetween
        ) {
            Text("左侧")
            Text("右侧")
        }
        
        // Spacer = 空白间距，类似 margin/padding
        Spacer(modifier = Modifier.height(16.dp))
        
        // Box = position: relative（层叠布局）
        Box(modifier = Modifier.fillMaxSize()) {
            Text("覆盖在底层")
        }
    }
}
```

### 2.6 修饰符（Modifier = 样式 + 布局）

```kotlin
Text(
    text = "Hello",
    modifier = Modifier
        .fillMaxWidth()           // 宽度撑满
        .padding(16.dp)          // 内边距
        .background(Color.Blue)   // 背景色
        .clickable { }           // 点击事件
)
```

**常见 Modifier：**
| Compose | CSS |
|---------|-----|
| `.padding(16.dp)` | `padding: 16px` |
| `.fillMaxWidth()` | `width: 100%` |
| `.size(48.dp)` | `width: 48px; height: 48px` |
| `.background(Color.Red)` | `background: red` |
| `.clickable { }` | `onclick` |
| `.border(1.dp, Color.Black)` | `border: 1px solid black` |

### 2.7 列表渲染（对应 v-for）

```kotlin
// LazyColumn = virtualized list（高性能）
LazyColumn(
    verticalArrangement = Arrangement.spacedBy(8.dp)
) {
    items(accounts) { account ->
        AccountCard(account = account)
    }
}

// 懒加载 + key
items(
    items = accounts,
    key = { it.id }
) { account ->
    AccountCard(account = account)
}
```

**对比 Vue：**
```vue
<div v-for="account in accounts" :key="account.id">
    <AccountCard :account="account" />
</div>
```

### 2.8 导航/页面切换

```kotlin
// MainActivity 中管理屏幕状态
var currentScreen by remember { mutableStateOf("main") }

when (currentScreen) {
    "main" -> MainScreen(onNavigateToSettings = { currentScreen = "settings" })
    "settings" -> SettingsScreen(onBack = { currentScreen = "main" })
}
```

**对比 Vue Router：**
```vue
<template>
    <MainScreen v-if="route === 'main'" />
    <SettingsScreen v-else-if="route === 'settings'" />
</template>
```

## 第三章：ViewModel 和状态管理

### 3.1 ViewModel 基本结构

```kotlin
// 定义 UI 状态
data class MainUiState(
    val accounts: List<Account> = emptyList(),
    val isLoading: Boolean = false,
    val isRefreshing: Boolean = false
)

// ViewModel = 状态管理器
class MainViewModel(
    private val repository: AccountRepository  // 依赖注入
) : ViewModel() {
    
    // 私有可变状态
    private val _uiState = MutableStateFlow(MainUiState())
    
    // 对外暴露不可变状态
    val uiState: StateFlow<MainUiState> = _uiState.asStateFlow()
    
    // 方法处理业务逻辑
    fun refreshAccounts() {
        viewModelScope.launch {
            _uiState.update { it.copy(isLoading = true) }
            // 执行操作...
            _uiState.update { it.copy(isLoading = false) }
        }
    }
}
```

**对比 Vue：**
```typescript
// store.ts (Pinia)
export const useMainStore = defineStore('main', {
    state: () => ({
        accounts: [],
        isLoading: false
    }),
    actions: {
        async refreshAccounts() {
            this.isLoading = true
            // ...
            this.isLoading = false
        }
    }
})
```

### 3.2 在 Compose 中使用 ViewModel

```kotlin
@Composable
fun MainScreen(viewModel: MainViewModel) {
    // collectAsState = 将 Flow 转为 Compose 状态
    val uiState by viewModel.uiState.collectAsState()
    
    // 使用状态
    if (uiState.isLoading) {
        CircularProgressIndicator()
    }
    
    // 调用方法
    Button(onClick = { viewModel.refreshAccounts() }) {
        Text("刷新")
    }
}
```

**在 Activity 中创建：**
```kotlin
val viewModel: MainViewModel = viewModel {
    MainViewModel(repository, quotaService)
}
```

## 第四章：DataStore 数据持久化

### 4.1 DataStore 基础

```kotlin
// 创建 DataStore
val Context.dataStore by preferencesDataStore(name = "settings")

// 写入数据
suspend fun saveDarkMode(enabled: Boolean) {
    dataStore.edit { preferences ->
        preferences[PreferencesKeys.DARK_MODE] = enabled
    }
}

// 读取数据
val darkModeFlow: Flow<Boolean> = dataStore.data.map { preferences ->
    preferences[PreferencesKeys.DARK_MODE] ?: false
}
```

**对比前端：**
```javascript
// localStorage
localStorage.setItem('darkMode', JSON.stringify(true))
const darkMode = JSON.parse(localStorage.getItem('darkMode'))

// DataStore 更适合复杂对象，且是异步的
```

```kotlin
class AccountRepository(private val context: Context) {

        Json.decodeFromString(json)
    }
    
    suspend fun saveAccount(account: UserAccount) {
        // 读取 -> 修改 -> 保存
    }
}
```

## 第五章：网络请求

### 5.1 Kotlin Coroutines 异步

```kotlin
// suspend 函数 = async 函数
suspend fun fetchData(): Result<Data> {
    return withContext(Dispatchers.IO) {
        // IO 线程执行网络请求
        try {
            val response = api.call()
            Result.success(response)
        } catch (e: Exception) {
            Result.failure(e)
        }
    }
}

// 调用方
viewModelScope.launch {
    val result = fetchData()
    result.onSuccess { data -> ... }
    result.onFailure { error -> ... }
}
```

**对比 JavaScript：**
```javascript
async function fetchData() {
    try {
        const response = await api.call()
        return { success: true, data: response }
    } catch (error) {
        return { success: false, error }
    }
}
```

### 5.2 使用 HttpURLConnection

```kotlin
suspend fun queryQuota(account: UserAccount): Result<QuotaData> = withContext(Dispatchers.IO) {
    try {
        val url = URL("http://api.example.com/query")
        val connection = url.openConnection() as HttpURLConnection
        connection.requestMethod = "POST"
        connection.doOutput = true
        connection.connectTimeout = 15000
        
        // 发送请求体
        val jsonBody = JSONObject().apply {
            put("username", account.username)
            put("token", account.token)
        }.toString()
        
        OutputStreamWriter(connection.outputStream).use { it.write(jsonBody) }
        
        // 读取响应
        val responseCode = connection.responseCode
        val reader = BufferedReader(InputStreamReader(connection.inputStream))
        val response = reader.readText()
        reader.close()
        
        // 解析 JSON
        val json = JSONObject(response)
        if (json.getBoolean("success")) {
            Result.success(parseData(json))
        } else {
            Result.failure(Exception(json.getString("message")))
        }
    } catch (e: Exception) {
        Result.failure(e)
    }
}
```

## 推荐学习路径

1. **Kotlin 基础**（1天）
   - 刷一遍 Kotlin 官方教程：https://kotlinlang.org/docs/getting-started.html

2. **Compose 入门**（2-3天）
   - 官方 Codelab：https://developer.android.com/codelabs/jetpack-compose-basics

3. **ViewModel + StateFlow**（1天）
   - 官方文档：https://developer.android.com/topic/libraries/architecture/viewmodel
   - 重点：状态提升、副作用处理

4. **DataStore**（1天）
   - 官方文档：https://developer.android.com/topic/libraries/architecture/datastore
   - 重点：Repository 模式

5. **实践项目**
   - 克隆本项目，用 Android Studio 打开
   - 尝试修改 UI、增加功能
   - 参考 Google I/O 应用源码
